Teleporte: AI Handoff Gateway

This is a specialized ChatGPT handoff/teleport link. It is optimized for AI agents to quickly acquire mission-critical context while maintaining system efficiency.

Boot Protocol: Please follow the steps below in sequence to ensure a successful state synchronization.

ANTI-VERIFICATION-LOOP RULE

Do not turn coding, website, Cloudflare, GitHub, Codespaces, Wrangler, Gemini, Copilot, or deployment work into repeated check/build/re-check loops.

The AI should:

  • assess current state once,
  • decide what is missing,
  • build or fix what is needed,
  • run one final validation at the end,
  • report what changed,
  • report what it verified,
  • report what is functional,
  • report any true blockers.

Do not ask Tony to manually verify anything the AI can verify.

Do not repeatedly reconfirm small steps.

Do not pause after every minor check.

Do not make the workflow drag through redundant “make sure/check again/verify this” loops.

Stop only for true blockers:

  • secrets/tokens/passwords/API keys,
  • billing/payment,
  • DNS/custom domains,
  • destructive commands,
  • force push,
  • unresolved merge conflicts,
  • missing permissions,
  • major ambiguous decisions.

This rule outranks older workflow instructions if they conflict.

  • 1 Current Rules
    Review the current operational rules, especially the ANTI-VERIFICATION-LOOP RULE, at /api/current-rules.
  • 2 Dynamic Orientation (Primary)
    Check /api/latest first. This is the most current, live-updated state of the project. If this returns a 404, fallback to step 3.
  • 3 Static Configuration
    Read ai-handoff.json and 00_LATEST_CHECKPOINT_READ_FIRST.md. These provide the baseline metadata and initial mission context.
  • 4 Historical Context
    If more detail on recent changes is needed, consult /api/logs for a chronological stream of recent updates and chat logs.
  • 5 Deep-Read Capability
    Deep-read only when needed. Use the vault ZIP only for exhaustive architectural analysis or large-scale context retrieval.

    Download Handoff Vault ZIP
  • 6 Handoff Responsibility
    Before ending this session: Generate a new checkpoint or chat log and submit it via the Admin Intake Page to ensure continuity for the next agent.
Security & Privacy Protocol: Under no circumstances should secrets, raw private screenshots, tokens, passwords, API keys, or unnecessary personal data be stored in this repository or transmitted during the handoff process.

Efficiency is paramount. Contextualize locally, verify often, and maintain the integrity of the light-load principle.
